# Shruti Shejul Portfolio

This folder contains a professionally organized version of the portfolio project.

## Project Structure

- `index.html` — Main HTML entry point with semantic section comments and SEO/meta tags.
- `css/style.css` — Core styling, layout, colors, typography, and section rules.
- `css/animations.css` — Animation keyframes and reveal-on-scroll effects.
- `css/responsive.css` — Mobile and tablet breakpoints.
- `js/main.js` — Cursor, navbar scroll, project hover effects, reveal observer, and form submission.
- `js/particles.js` — Floating particle background in the hero section.
- `js/animations.js` — 3D hero object animation.
- `assets/icons/favicon.svg` — Favicon for browser tabs.
- `assets/images/` — Placeholder location for future image assets.
- `assets/icons/` — Icons and visual assets.
- `assets/resume/` — Resume downloads and related documents.

## How the Files Connect

1. `index.html` loads the stylesheet files and script files.
2. `css/style.css` defines the look and layout.
3. `css/animations.css` adds motion and reveal effects.
4. `css/responsive.css` adjusts layout for smaller screens.
5. `js/main.js` handles user interactions.
6. `js/particles.js` and `js/animations.js` animate the hero section.

## File Responsibilities

### HTML
- `index.html` contains all page content and section comments:
  - Navbar
  - Hero
  - About
  - Skills
  - Projects
  - Resume
  - Contact
  - Footer

### CSS
- `css/style.css` keeps the core design and sections intact.
- `css/animations.css` stores animation logic.
- `css/responsive.css` stores responsive behavior.

### JavaScript
- `js/main.js` contains general DOM interactions.
- `js/particles.js` powers the hero particles.
- `js/animations.js` powers the 3D object.

## Notes

- No design, animation, content, layout, or functionality was intentionally changed.
- The structure is organized for real-world development workflows and future expansion.
